  • 스크롤을 없애고 싶은데...
  • 이번 프로젝트는 모바일 전용 페이지로, 가로 414px에 맞춰 제작했습니다. 핸드폰 목업 프레임을 고정시키고 스크롤하면 가려진 부분들이 올라와 실제 핸드폰에서 보는 것처럼 구현하고자 했습니다. 그런데 목업 프레임에 스크롤이 생겨서 사라지지 않았습니다. css로 구현하면 .box::-webkit-scrollbar{display:none;}를 쓰면되는데, 리액트에서는 이런 구문이 허용되지 않았습니다. 리액트에서 ::를 쓸 수 있는 방법을 찾아보고자 검색을 시작했고, 스타일드 컴포넌트로 box를 만든 다음 {}안에 &::를 넣어 사용하면 된다는 것을 알게 되어 문제를 해결할 수 있었습니다.
  • build 파일이 안열린다?
  • 프로젝트 제작을 완료하고, npm run build를 실행해 빌드 파일을 생성했습니다. 빌드파일을 깃허브에 git push까지 마치고 작동하길 기다리는데 한참을 기다려도 반응이 없었습니다. 리액트 파일을 다시 생성해보기도 하고 코드를 몇 번 갈아엎던 중, 학원에서 index 파일에 경로를 수정해야 한다는 걸 배운 것을 깨닫게 되었습니다. 파일을 열여보니 경로가 /static/으로 된 부분을 ./static으로 변경해주어 문제를 해결했습니다. 매번 .을 찍기 번거롭다 생각되어 구글로 검색해보았고, package.json에서 "homepage": "./"를 추가하면 수정없이 사용할 수 있는 것도 알게됐습니다.

  • Review
  • 이번 작업은 전체 페이지를 리액트로 구성했습니다. data파일와 레이아웃/디자인 파일로 분리해 컴포넌트를 생성했습니다. 메인 콘텐츠 영역이 "ul>li>사진+h2태그+p태그"로 구성되기 때문에 ul/li를 채울 각각의 레이아웃용 컴포넌트를 만들고, data파일을 불러와 구성했습니다. 코딩을 하면서 절실히 CSS를 쓰고 싶은 순간이 있었으나 리액트로만 해보자는 생각으로 작업했습니다. hover나 토글 버튼처럼 css나 자바스크립트로 쉽게 구현되는 것들도 리액트에서는 어렵기만 했습니다. 막막함을 해결하기 위해 수업 이후 2~3시간 정도 개발 관련 유튜브를 보며 리액트에 대한 기초를 좀 더 탄탄하게 다지고자 노력했습니다. 학원에서 배우지 못했던 useState, useEffect와 같은 hook에 대해서도 공부할 수 있었습니다. 앞으로 좀 더 공부하여 hover 구현과 select태그 페이지 이동 등 완전하게 작동할 수 있는 사이트를 구현해보고자 합니다.
