  • 원하는 모양대로 svg 파일 생성이 안 된다?
  • 일러스트에서 png 파일을 만들어 컨버터를 이용해 svg파일을 생성했습니다. 그런데 끝부분이 둥글게 변환되는 문제가 발생했습니다. 다른 사이트에서 여러 번 시도했지만 동일한 형태로 변환되었습니다. 이에 path generator를 이용해 직접 svg파일을 만들자 생각해 Move, Line, Arc 등 svg의 기본적인 개념을 명확하게 익히는 시간을 가졌습니다. 그렇게 제작을 하던 중, 부채꼴을 만드는 데 난관에 봉착했습니다. Arc, line으로는 쉽게 그려지지 않았던 것입니다. MDN 사이트를 확인하여, circle로 svg를 생성하고 cx, cy, r 값을 지정하면 부채꼴 모양을 만들 수 있다는 것을 확인해 문제를 해결했습니다. 이외에 svg 그룹화시켜주는 g, 여러 개의 점을 연결하는 직선 polyline 등 svg에 깊이 있게 이해하는 시간을 가질 수 있었습니다.
  • SCSS로 컴파일한 css의 레이아웃이 깨진다?
  • 반응형 페이지를 제작하다 보니, css를 수정할 때 마구 쌓아둔 짐을 뒤지는 것처럼 복잡한 느낌이 들었습니다. 이에 SCSS를 활용해 페이지를 다시 제작해보기로 했습니다. setting.json에 scss를 css로 자동 저장해주는 기능과 터미널에서 명령행으로 컴파일하는 방법도 익혔습니다. 그렇게 다시 파일을 저장했는데, 레이아웃이 깨져서 나타났습니다. 중첩을 잘못 넣었나 싶어 SCSS파일을 다시 제작해보기도 했습니다. 그러던 중, container의 width 값에 오류가 있는 걸 확인했습니다. 띄어쓰기를 잘못하여 calc (100%- 6px)로 브라우저가 해석하지 못하는 값을 준 것입니다. 다시 기존 css를 연결하여 페이지를 열어보니 개발자 도구에서 잘못된 속성 값이라고 하여 브라우저가 값을 무효화한 것을 확인할 수 있었습니다. 개발자 도구의 유용함을 다시 한번 깨닫는 경험이었습니다.
  • CHALLENGE
  • Review
  • 이번 페이지는 svg 애니메이션과 그리드 라인을 주제로 제작했습니다. 레이아웃 배치가 다소 엉성한 느낌이 들어 아쉬운 감이 있지만 stroke-dashoffset, stroke-dasharray 등을 사용해보기도 하고, SCSS, svg 제작 등 다양한 도전을 해본 페이지입니다. 그리고 이 사이트는 처음으로 SCSS를 사용해 작업하여 의미가 있었습니다. extension 설치부터 컴파일, 중첩, mixin, 변수 사용 방법을 유튜브를 보며 4시간 만에 습득했습니다. css 파일을 수정할 때 찾는 것이 번거로울 때가 있었는데, SCSS를 사용하니 중첩의 기능이 너무나도 편리하게 느껴졌습니다. 새로운 내용이 많아 어려운 작업이었지만 그만큼 많은 공부가 된 작업이었습니다. 이번 작업을 시작으로 SCSS의 if, 반복문 등 다량의 데이터에도 효율적으로 활용할 수 있도록 좀 더 깊이 있게 공부할 예정입니다.
